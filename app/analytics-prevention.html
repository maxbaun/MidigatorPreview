Completed Prevention Cases

This table contains in-depth information about each prevention case that has been completed and resolved.

To view alerts that have not been completed and still need to be resolved, please visit the Manager or Workflow pages.

Column Descriptions

Prevention case type
The vendor or service which issued the prevention case

Case ID
Unique case identifier often provided by the prevention service provdier (Ehoca, Verifi, Order Insight, Eliminator)

Order ID
Unique identifier for the CRM order associated with the alert

Received date
The date when Midigator received the prevention case from the provider

Transaction date
Date when the original transaction was processed

First name
The first name of the customer disputing the transaction

Last name
The last name of the customer disputing the transaction

Amount
The amount of the original transaction

Currency
The currency used in the original transaction

MID
Merchant ID - Unique identifier for the Merchant: it is used to process credit cards

MID alias
The common name given to the MID when it was created within Midigator. Used to easily identify MIDs.

BIN
First six digits of number of the credit card used in the original transaction

Last 4
Last four numbers of the credit card used in the original transaction

ARN
The Acquirer Reference Number associated with the transaction (if available)

Descriptor
The descriptor (as seen on the customer's bank statement) associated with the MID through which the transaction was run.

Prevention case resolution
The final resolution statement for the individual prevention case.

Prevention Case Resolutions

The following prevention case resolutions are available for each prevention case type. Depending on your integration level, these values may be self-reported or automatically performed by the system.

Ethoca Alerts
  • Full Refund: I issued a full refund on the transaction
  • Partial Refund: I refunded the remaining amount of the transaction
  • Already Refunded: The transaction was refunded prior to receiving the alert
  • Duplicate Alert: The transaction has already received another alert.
  • Already a Chargeback: The transaction received a chargeback prior to the alert
  • Failed Authorization: The transaction was declined and no refund is required
  • 3DS Alert: The transaction has a 3D secure code and no refund is required
  • Not Found: I am unable to locate the transaction
  • Refund Error: The refund was declined on this transaction
  • Expired: The alert was not resolved in time and therefore expired [System]
